# DynNav: Dynamic Navigation & Rerouting in Unknown Environments

## Ελληνική τεκμηρίωση για κατανόηση του project

Το **DynNav** είναι ένα ερευνητικό framework για αυτόνομη πλοήγηση και δυναμικό επανασχεδιασμό διαδρομής σε άγνωστα ή μεταβαλλόμενα περιβάλλοντα. Ο στόχος του project δεν είναι απλώς να υπολογίσει μια διαδρομή από ένα σημείο σε ένα άλλο. Ο στόχος είναι να μελετήσει τι συμβαίνει όταν ένα ρομπότ πρέπει να κινηθεί ενώ οι πληροφορίες που διαθέτει δεν είναι τέλειες: ο χάρτης μπορεί να είναι ελλιπής, οι αισθητήρες μπορεί να έχουν θόρυβο, η θέση του ρομπότ μπορεί να είναι αβέβαιη, άλλα ρομπότ μπορεί να διαφωνούν μεταξύ τους και ένα learning-based σύστημα μπορεί να κάνει λάθος.

Το README αυτό γράφτηκε για να εξηγεί το project στα ελληνικά με τρόπο κατανοητό και για κάποιον που δεν έχει προηγούμενη εμπειρία στη ρομποτική. Παράλληλα, κρατά επιστημονικό ύφος, ώστε να φαίνεται καθαρά ποιο ήταν το ερευνητικό σκεπτικό πίσω από κάθε module.

# 1. Τι προσπαθεί να λύσει το DynNav

Σε ένα απλό παράδειγμα πλοήγησης, ένα ρομπότ γνωρίζει τον χάρτη, γνωρίζει την αρχική του θέση, γνωρίζει τον στόχο και χρησιμοποιεί έναν planner για να βρει μια διαδρομή. Αυτό όμως είναι ένα ιδανικό σενάριο. Στην πράξη, ένα ρομπότ σπάνια έχει τέλεια πληροφορία.

Το DynNav εξετάζει την πλοήγηση ως πρόβλημα λήψης αποφάσεων υπό αβεβαιότητα. Αυτό σημαίνει ότι το ρομπότ δεν πρέπει να αποφασίζει μόνο με βάση το ποια διαδρομή είναι μικρότερη, αλλά και με βάση το πόσο αξιόπιστες είναι οι πληροφορίες του, πόσο επικίνδυνη είναι κάθε διαδρομή και αν υπάρχει τρόπος να διορθωθεί μια κακή απόφαση.

Παραδείγματα προβλημάτων που εξετάζει το project:

- το ρομπότ δεν γνωρίζει με απόλυτη ακρίβεια πού βρίσκεται,
- οι αισθητήρες μπορεί να δίνουν θορυβώδεις ή λανθασμένες μετρήσεις,
- ένα εμπόδιο μπορεί να εμφανιστεί μετά τον αρχικό σχεδιασμό,
- η πιο σύντομη διαδρομή μπορεί να είναι πιο επικίνδυνη,
- το ρομπότ μπορεί να εγκλωβιστεί σε περιοχή χωρίς ασφαλή επιστροφή,
- πολλά ρομπότ μπορεί να χρειάζεται να συντονιστούν,
- κάποιο robot σε ένα swarm μπορεί να στέλνει κακή πληροφορία,
- ένα μοντέλο μηχανικής μάθησης μπορεί να αποτύχει σε σπάνιο σενάριο.

Η βασική ιδέα είναι ότι η ασφαλής πλοήγηση δεν είναι μόνο path planning. Είναι συνδυασμός perception, uncertainty estimation, risk-aware planning, safety constraints, learning, security και multi-robot coordination.

# 2. Γιατί το project χωρίζεται σε 26 contributions

Το DynNav χωρίζεται σε 26 research contributions επειδή κάθε contribution εξετάζει διαφορετικό σημείο αποτυχίας της αυτόνομης πλοήγησης. Αυτό κάνει το project modular: κάθε module μπορεί να μελετηθεί ξεχωριστά, να αξιολογηθεί με δικά του πειράματα και στη συνέχεια να συνδεθεί με την ευρύτερη αρχιτεκτονική.

Η λογική δεν είναι ότι κάθε module λύνει όλο το πρόβλημα. Αντίθετα, κάθε module απαντά σε μία συγκεκριμένη ερώτηση:

- Πώς μειώνουμε τον χρόνο του A*;
- Πώς εκτιμούμε την αβεβαιότητα της θέσης;
- Πώς σχεδιάζουμε με βάση τον κίνδυνο;
- Πώς αποφεύγουμε μη αναστρέψιμες αποφάσεις;
- Πώς περνά το robot σε safe mode;
- Πώς λαμβάνουμε υπόψη ενέργεια και συνδεσιμότητα;
- Πώς εξερευνούμε άγνωστες περιοχές;
- Πώς ανιχνεύουμε spoofing ή ανωμαλίες;
- Πώς συνεργάζονται πολλά robots;
- Πώς ενσωματώνονται LLMs/VLMs χωρίς να χάνεται ο έλεγχος;
- Πώς αξιολογείται η ανθεκτικότητα σε adversarial επιθέσεις;
- Πώς επιτυγχάνεται συναίνεση σε swarm με Byzantine agents;

Αυτή η δομή δείχνει ότι το project δεν είναι μία απλή υλοποίηση ενός planner, αλλά ένα ευρύτερο ερευνητικό πλαίσιο για robust autonomous navigation.

# 3. Συνολική αρχιτεκτονική

Η αρχιτεκτονική του DynNav μπορεί να γίνει κατανοητή ως στοίβα επιπέδων:

```text
ROS 2 Humble / TurtleBot3 / Gazebo / Nav2 / slam_toolbox
        ↓
Perception και Mapping
        ↓
Uncertainty Estimation
        ↓
Planning Core
        ↓
Risk-Aware και Belief-Space Planning
        ↓
Safety Layer
        ↓
Learning Layer
        ↓
Security και Multi-Robot Coordination
```

Σε πρακτικό επίπεδο, η βάση του project είναι το ROS 2 και το TurtleBot3/Gazebo περιβάλλον. Πάνω σε αυτή τη βάση οργανώνονται modules που αφορούν planning, perception, uncertainty, safety, learning και coordination.

Το σημαντικό σημείο είναι ότι τα modules δεν υπάρχουν τυχαία. Κάθε layer απαντά σε μια ανάγκη της πραγματικής πλοήγησης:

- το perception layer δίνει πληροφορία για τον κόσμο,
- το uncertainty layer εκτιμά πόσο αξιόπιστη είναι αυτή η πληροφορία,
- το planning layer υπολογίζει πιθανές διαδρομές,
- το risk layer αξιολογεί τις διαδρομές με βάση πιθανές αποτυχίες,
- το safety layer περιορίζει επικίνδυνες ενέργειες,
- το learning layer βελτιώνει την απόδοση ή την προσαρμοστικότητα,
- το security/coordination layer εξετάζει επιθέσεις, σφάλματα και συνεργασία πολλών robots.

## Contribution 01 — Learned A* Heuristics

### Πρόβλημα

Ο A* είναι ένας από τους πιο γνωστούς αλγορίθμους σχεδιασμού διαδρομής. Είναι αξιόπιστος και κατανοητός, αλλά σε μεγάλους χάρτες ή σε σενάρια όπου το robot πρέπει να κάνει συχνό replanning μπορεί να γίνει υπολογιστικά ακριβός. Το πρόβλημα δεν είναι ότι ο A* δεν βρίσκει διαδρομή, αλλά ότι μπορεί να χρειαστεί να εξερευνήσει πολλούς κόμβους πριν καταλήξει στη λύση.

Για ένα robot που κινείται σε πραγματικό χρόνο, αυτό είναι σημαντικό. Αν αλλάξει το περιβάλλον και ο planner αργήσει να υπολογίσει νέα διαδρομή, το robot μπορεί να συνεχίσει να κινείται με παλιά και πιθανώς μη ασφαλή πληροφορία.

### Τι υλοποιήθηκε

Στο contribution αυτό υλοποιείται learned heuristic για τον A*. Η ιδέα είναι να χρησιμοποιηθεί μάθηση ώστε η heuristic να οδηγεί την αναζήτηση πιο αποτελεσματικά. Ο κλασικός A* δεν αντικαθίσταται. Παραμένει η βάση του planner, αλλά υποστηρίζεται από learned πληροφορία που μειώνει τις άσκοπες επεκτάσεις κόμβων.

### Σημασία για το DynNav

Το DynNav χρειάζεται γρήγορο replanning, επειδή ασχολείται με άγνωστα και δυναμικά περιβάλλοντα. Αν ο χάρτης αλλάξει ή εμφανιστεί νέο εμπόδιο, το robot πρέπει να αντιδράσει γρήγορα. Η μείωση των node expansions βοηθά ακριβώς σε αυτό.

### Αποτελέσματα και ερμηνεία

Το project αναφέρει μείωση των node expansions περίπου κατά 35%. Αυτό σημαίνει ότι η learned heuristic βοηθά τον planner να εστιάσει σε πιο χρήσιμα τμήματα του χώρου αναζήτησης. Η βελτίωση είναι σημαντική γιατί μειώνει υπολογιστικό κόστος χωρίς να αλλάζει πλήρως η φιλοσοφία του κλασικού A*.

Ο βασικός περιορισμός είναι ότι ένα learned heuristic εξαρτάται από τα δεδομένα στα οποία έχει εκπαιδευτεί ή από τα σενάρια στα οποία έχει ρυθμιστεί. Αν το robot βρεθεί σε πολύ διαφορετικό περιβάλλον, η βελτίωση μπορεί να είναι μικρότερη. Για αυτό η προσέγγιση είναι χρήσιμη όταν συνδυάζεται με κλασικό planning και όχι όταν αντικαθιστά πλήρως τη δομή του planner.

## Contribution 02 — Uncertainty Estimation με EKF/UKF

### Πρόβλημα

Ένα robot δεν γνωρίζει ποτέ με απόλυτη ακρίβεια την κατάστασή του. Οι αισθητήρες έχουν θόρυβο, η odometry συσσωρεύει σφάλμα και η εκτίμηση θέσης μπορεί να αποκλίνει από την πραγματική θέση. Για έναν αρχάριο αναγνώστη, αυτό σημαίνει ότι το robot μπορεί να «νομίζει» πως βρίσκεται σε ένα σημείο ενώ στην πραγματικότητα βρίσκεται λίγο πιο δίπλα.

Αυτό είναι σοβαρό πρόβλημα. Αν ο planner σχεδιάσει διαδρομή με βάση λάθος θέση, μπορεί να περάσει πολύ κοντά από εμπόδιο ή να πάρει λάθος στροφή.

### Τι υλοποιήθηκε

Το module χρησιμοποιεί Extended Kalman Filter και Unscented Kalman Filter για εκτίμηση της κατάστασης υπό θόρυβο. Αντί το σύστημα να κρατά μόνο μία τιμή θέσης, εκτιμά και την αβεβαιότητα γύρω από αυτή την τιμή.

### Σημασία για το DynNav

Η αβεβαιότητα είναι κεντρική έννοια στο DynNav. Τα επόμενα modules, όπως belief-space planning και risk-aware planning, χρειάζονται πληροφορία όχι μόνο για το πού είναι το robot, αλλά και για το πόσο σίγουρο είναι το σύστημα για αυτή την εκτίμηση.

### Αποτελέσματα και ερμηνεία

Η αξιολόγηση του module δείχνει πώς EKF/UKF μπορούν να χρησιμοποιηθούν για noisy sensors. Η σημασία δεν είναι απλώς ότι παράγεται καλύτερη εκτίμηση θέσης, αλλά ότι η πληροφορία αβεβαιότητας γίνεται διαθέσιμη στα επόμενα στάδια απόφασης.

Ο περιορισμός είναι ότι τα φίλτρα αυτά βασίζονται σε υποθέσεις για το μοντέλο κίνησης και τον θόρυβο. Αν οι υποθέσεις αυτές δεν ισχύουν, η εκτίμηση μπορεί να γίνει λιγότερο αξιόπιστη.

## Contribution 03 — Belief-Space και Risk Planning

### Πρόβλημα

Πολλοί planners σχεδιάζουν σαν η κατάσταση του κόσμου να είναι πλήρως γνωστή. Αυτό όμως δεν ισχύει σε πραγματική πλοήγηση. Το robot έχει belief, δηλαδή μια εκτίμηση με αβεβαιότητα, όχι απόλυτη γνώση.

Επιπλέον, η μικρότερη διαδρομή δεν είναι πάντα ασφαλής. Μια διαδρομή μπορεί να είναι σύντομη αλλά να περνά από περιοχή όπου υπάρχει μεγάλη πιθανότητα αποτυχίας.

### Τι υλοποιήθηκε

Το module εισάγει belief-space planning και risk-weighted A* με CVaR. Το CVaR δίνει έμφαση στις χειρότερες περιπτώσεις, όχι μόνο στο μέσο κόστος.

### Σημασία για το DynNav

Αυτό είναι βασικό για ένα risk-sensitive navigation framework. Το robot δεν επιλέγει απλώς τη διαδρομή με το μικρότερο μήκος, αλλά λαμβάνει υπόψη πιθανές επικίνδυνες εκβάσεις.

### Αποτελέσματα και ερμηνεία

Η προσέγγιση επιτρέπει πιο συντηρητικές αλλά ασφαλέστερες επιλογές όταν υπάρχει υψηλή αβεβαιότητα. Αυτό είναι χρήσιμο σε περιβάλλοντα όπου μια σπάνια αποτυχία μπορεί να έχει μεγάλο κόστος.

Ο περιορισμός είναι ότι risk-aware planning μπορεί να οδηγήσει σε μεγαλύτερες ή πιο συντηρητικές διαδρομές. Αυτό όμως είναι συχνά αποδεκτό σε safety-critical robotics.

## Contribution 04 — Irreversibility και Returnability

### Πρόβλημα

Σε άγνωστο περιβάλλον, ένα robot μπορεί να πάρει μια απόφαση που φαίνεται χρήσιμη τοπικά, αλλά το οδηγεί σε κατάσταση από την οποία δεν μπορεί εύκολα να επιστρέψει. Για παράδειγμα, μπορεί να μπει σε στενό πέρασμα, dead-end ή περιοχή χαμηλής πληροφορίας.

### Τι υλοποιήθηκε

Το module εισάγει returnability constraints. Κάθε απόφαση αξιολογείται και ως προς το αν το robot διατηρεί δυνατότητα επιστροφής ή ανάκαμψης.

### Σημασία για το DynNav

Η returnability είναι σημαντική για εξερεύνηση άγνωστων χώρων. Ένα robot δεν πρέπει μόνο να προχωρά προς τον στόχο, αλλά και να αποφεύγει καταστάσεις που μειώνουν τις μελλοντικές επιλογές του.

### Αποτελέσματα και ερμηνεία

Το module δείχνει πώς οι returnability περιορισμοί μπορούν να μειώσουν τον κίνδυνο dead-ends. Η συμπεριφορά γίνεται πιο προσεκτική, επειδή ο planner σκέφτεται όχι μόνο το επόμενο βήμα, αλλά και το αν θα υπάρχει ασφαλής συνέχεια.

Περιορισμός: το robot μπορεί να απορρίψει διαδρομές που είναι γρήγορες αλλά λιγότερο αναστρέψιμες. Αυτό αυξάνει τη συντηρητικότητα, αλλά βελτιώνει την ασφάλεια.

## Contribution 05 — Safe-Mode Navigation

### Πρόβλημα

Το robot δεν πρέπει να χρησιμοποιεί πάντα την ίδια στρατηγική. Όταν ο κίνδυνος είναι χαμηλός, μπορεί να κινηθεί πιο αποδοτικά. Όταν ο κίνδυνος αυξηθεί, πρέπει να γίνει πιο προσεκτικό.

### Τι υλοποιήθηκε

Το module υλοποιεί adaptive risk-triggered safe mode. Όταν συγκεκριμένοι δείκτες κινδύνου ξεπεράσουν όριο, το robot περνά σε πιο συντηρητική λειτουργία.

### Σημασία για το DynNav

Το safe mode λειτουργεί ως πρακτικός μηχανισμός προστασίας. Δεν απαιτεί να αποτύχει πρώτα το σύστημα. Ανιχνεύει αυξημένο κίνδυνο και αλλάζει συμπεριφορά πριν η κατάσταση γίνει επικίνδυνη.

### Αποτελέσματα και ερμηνεία

Η προσέγγιση δείχνει ότι η πλοήγηση μπορεί να προσαρμόζεται δυναμικά στο επίπεδο κινδύνου. Αυτό είναι σημαντικό σε περιβάλλοντα που αλλάζουν, επειδή η ίδια πολιτική δεν είναι κατάλληλη για όλες τις συνθήκες.

## Contribution 06 — Energy και Connectivity Aware Planning

### Πρόβλημα

Ένα robot έχει περιορισμένη μπαταρία και μπορεί να χρειάζεται σύνδεση με δίκτυο ή με άλλα robots. Ένας planner που κοιτά μόνο την απόσταση μπορεί να επιλέξει διαδρομή που είναι σύντομη αλλά ενεργειακά κακή ή επικοινωνιακά μη ασφαλής.

### Τι υλοποιήθηκε

Το module εισάγει constraints για battery και WiFi/connectivity στο planning.

### Σημασία για το DynNav

Σε πραγματικές αποστολές, το robot πρέπει να ολοκληρώσει την αποστολή χωρίς να εξαντλήσει την ενέργεια και χωρίς να χαθεί η δυνατότητα συντονισμού. Αυτό κάνει το planning πιο ρεαλιστικό.

### Αποτελέσματα και ερμηνεία

Το module δείχνει ότι ο σχεδιασμός διαδρομής μπορεί να επεκταθεί πέρα από γεωμετρικά κριτήρια. Η απόσταση δεν είναι το μόνο κόστος. Η ενέργεια και η συνδεσιμότητα είναι επίσης λειτουργικοί περιορισμοί.

## Contribution 07 — Next-Best-View Exploration

### Πρόβλημα

Όταν ο χάρτης είναι άγνωστος, το robot δεν γνωρίζει ποια περιοχή πρέπει να εξερευνήσει πρώτα. Αν κινηθεί τυχαία, μπορεί να χάσει χρόνο και ενέργεια.

### Τι υλοποιήθηκε

Το module χρησιμοποιεί information gain για να επιλέξει next-best-view, δηλαδή την επόμενη θέση που αναμένεται να προσφέρει τη μεγαλύτερη πληροφορία για τον χάρτη.

### Σημασία για το DynNav

Η εξερεύνηση είναι απαραίτητη σε άγνωστα περιβάλλοντα. Το robot πρέπει να επιλέγει κινήσεις που μειώνουν την άγνοια για τον κόσμο, όχι μόνο κινήσεις που φαίνονται κοντά στον στόχο.

### Αποτελέσματα και ερμηνεία

Η προσέγγιση οδηγεί σε πιο στοχευμένη χαρτογράφηση. Το robot προτιμά περιοχές που αποκαλύπτουν χρήσιμη πληροφορία, άρα βελτιώνει μελλοντικές αποφάσεις planning.

## Contribution 08 — Security και IDS

### Πρόβλημα

Ένα autonomous system μπορεί να δεχτεί λανθασμένα αισθητηριακά δεδομένα είτε λόγω βλάβης είτε λόγω επίθεσης. Αν το robot εμπιστεύεται τυφλά όλα τα δεδομένα, μπορεί να πάρει επικίνδυνη απόφαση.

### Τι υλοποιήθηκε

Το module χρησιμοποιεί χ² και CUSUM anomaly detection για ανίχνευση sensor spoofing ή ασυνήθιστης συμπεριφοράς.

### Σημασία για το DynNav

Η ασφάλεια δεν αφορά μόνο την αποφυγή εμποδίων. Αφορά και την αξιοπιστία των δεδομένων που χρησιμοποιεί το robot. Ένα IDS προσθέτει επίπεδο ελέγχου πριν τα λανθασμένα δεδομένα επηρεάσουν τον planner.

### Αποτελέσματα και ερμηνεία

Η ανίχνευση ανωμαλιών επιτρέπει στο σύστημα να αναγνωρίζει μετρήσεις που αποκλίνουν από την αναμενόμενη συμπεριφορά. Αυτό βοηθά ιδιαίτερα σε σενάρια spoofing.

## Contribution 09 — Multi-Robot Coordination

### Πρόβλημα

Όταν πολλά robots κινούνται στον ίδιο χώρο, δεν αρκεί κάθε robot να σχεδιάζει μόνο του. Αν δύο robots επιλέξουν την ίδια περιοχή ή διασταυρούμενες διαδρομές, μπορεί να δημιουργηθούν συγκρούσεις.

### Τι υλοποιήθηκε

Το module υλοποιεί decentralized conflict-free path allocation.

### Σημασία για το DynNav

## Contribution 18 — Formal Safety Shields

### Πρόβλημα

Ένας planner ή ένα learning model μπορεί να προτείνει εντολή που φαίνεται χρήσιμη αλλά παραβιάζει περιορισμούς ασφαλείας. Σε πραγματικό robot, ακόμη και μία επικίνδυνη εντολή μπορεί να οδηγήσει σε σύγκρουση.

### Τι υλοποιήθηκε

Το module χρησιμοποιεί STL monitor και CBF command filter. Το STL παρακολουθεί αν ικανοποιούνται χρονικοί κανόνες ασφαλείας, ενώ το CBF φιλτράρει εντολές ώστε να παραμένουν εντός ασφαλούς περιοχής.

### Σημασία για το DynNav

Το safety shield λειτουργεί ως τελευταίο επίπεδο προστασίας. Δεν εμπιστεύεται τυφλά τον planner ή το learning model. Ελέγχει την εντολή πριν αυτή εκτελεστεί.

### Αποτελέσματα και ερμηνεία

| Metric | Χωρίς shield | Με STL+CBF shield |
|---|---:|---:|
| Constraint violations / episode | 4.2 avg | 0.3 avg |
| Path length overhead | — | < 8% |
| Avg command correction | — | 0.026 m/s |

Η μείωση από 4.2 σε 0.3 violations ανά επεισόδιο είναι σημαντική. Δείχνει ότι το shield περιορίζει αποτελεσματικά επικίνδυνες ενέργειες. Το path overhead κάτω από 8% δείχνει ότι η αύξηση ασφάλειας δεν έρχεται με υπερβολικό κόστος στη διαδρομή.

Ο περιορισμός είναι ότι ένα shield εξαρτάται από το πώς έχουν οριστεί οι safety constraints. Αν οι περιορισμοί είναι ελλιπείς, το shield δεν μπορεί να προστατεύσει από κινδύνους που δεν έχουν μοντελοποιηθεί.

## Contribution 26 — Swarm Consensus

### Πρόβλημα

Σε ένα swarm, πολλά robots πρέπει να συμφωνήσουν σε κοινό σχέδιο. Αν κάποια robots είναι Byzantine, δηλαδή στέλνουν λανθασμένες ή κακόβουλες πληροφορίες, μια απλή πλειοψηφία μπορεί να οδηγήσει σε λάθος επιλογή.

### Τι υλοποιήθηκε

Το module χρησιμοποιεί Byzantine fault-tolerant plan consensus με weighted median.

### Σημασία για το DynNav

Η multi-robot autonomy απαιτεί ανθεκτικότητα σε agents που αποτυγχάνουν ή συμπεριφέρονται λανθασμένα. Το σύστημα πρέπει να επιλέγει σωστό σχέδιο ακόμη και όταν μέρος της πληροφορίας είναι κακό.

### Αποτελέσματα και ερμηνεία

| Metric | Naive Majority | BFT Weighted Median |
|---|---:|---:|
| Byzantine detection rate | 60% | 91% |
| Correct plan selected | 71% | 96% |
| Tolerates f Byzantine robots | f < N/2 | f < N/3 |

Τα αποτελέσματα δείχνουν σημαντική αύξηση στο correct plan selection και στην ανίχνευση Byzantine agents. Αυτό συμβαίνει επειδή το weighted median περιορίζει την επίδραση ακραίων ή ύποπτων προτάσεων. Έτσι το swarm γίνεται πιο robust σε λανθασμένη πληροφορία.

# 6. Συνολική ερμηνεία αποτελεσμάτων

Τα αποτελέσματα του DynNav δεν πρέπει να διαβαστούν σαν ένα ενιαίο benchmark με μία μόνο μετρική. Κάθε contribution αξιολογεί διαφορετική πλευρά της αυτόνομης πλοήγησης.

Τα πιο χαρακτηριστικά συμπεράσματα είναι:

- το Learned A* δείχνει ότι η μάθηση μπορεί να βελτιώσει κλασικούς planners χωρίς να τους αντικαταστήσει,
- το EKF/UKF uncertainty estimation δείχνει ότι η αβεβαιότητα πρέπει να μεταφέρεται στα επόμενα στάδια απόφασης,
- το risk planning δείχνει ότι η μικρότερη διαδρομή δεν είναι πάντα η καλύτερη,
- τα safety shields δείχνουν ότι formal constraints μπορούν να μειώσουν δραστικά τις παραβιάσεις ασφαλείας,
- το federated learning δείχνει ότι distributed learning μπορεί να πλησιάσει centralized performance,
- το curriculum RL δείχνει ότι η σταδιακή εκπαίδευση είναι πολύ σημαντική για δύσκολες navigation tasks,
- το BFT swarm consensus δείχνει ότι η απλή πλειοψηφία δεν αρκεί όταν υπάρχουν Byzantine agents.

Το κοινό συμπέρασμα είναι ότι η ασφαλής πλοήγηση χρειάζεται πολλά επίπεδα προστασίας και απόφασης. Δεν αρκεί ένας planner. Χρειάζεται uncertainty, risk, safety, learning, security και coordination.

# 7. Συμπέρασμα

Το DynNav παρουσιάζει μια ολοκληρωμένη ερευνητική προσέγγιση για dynamic navigation και rerouting σε άγνωστα περιβάλλοντα. Η αξία του project δεν βρίσκεται μόνο στο ότι περιλαμβάνει 26 modules, αλλά στο ότι κάθε module αντιστοιχεί σε ένα πραγματικό πρόβλημα της αυτόνομης πλοήγησης.

Το project δείχνει ότι ένα robot για να κινηθεί αξιόπιστα στον πραγματικό κόσμο πρέπει να κάνει περισσότερα από το να βρει μια διαδρομή. Πρέπει να εκτιμά αβεβαιότητα, να αποφεύγει κίνδυνο, να ελέγχει την ασφάλεια των εντολών του, να αντέχει σε λάθη αισθητήρων, να συνεργάζεται με άλλα robots και να μπορεί να εξηγεί αποτυχίες.

Με αυτή την έννοια, το DynNav είναι ένα framework που ενώνει κλασική ρομποτική, πιθανοτική εκτίμηση, machine learning, formal safety και distributed decision making σε ένα ενιαίο ερευνητικό περιβάλλον.
