# 1. Περίληψη

Το **DynNav** είναι ένα μεγάλο ερευνητικό project πάνω στην αυτόνομη πλοήγηση ρομπότ σε περιβάλλοντα που δεν είναι πλήρως γνωστά, δεν είναι στατικά και δεν είναι πάντα αξιόπιστα.

Το βασικό ερώτημα του έργου είναι:

> **Πώς πρέπει να κινηθεί ένα ρομπότ όταν δεν μπορεί να εμπιστευτεί απόλυτα τον χάρτη, τους αισθητήρες, τα άλλα ρομπότ ή ακόμη και τα learning-based μοντέλα που χρησιμοποιεί;**

Σε ένα απλό παράδειγμα πλοήγησης, ένα ρομπότ ξεκινά από ένα σημείο, γνωρίζει τον χάρτη, γνωρίζει τον στόχο και υπολογίζει μια διαδρομή. Αυτό όμως είναι ιδανική περίπτωση. Στον πραγματικό κόσμο τα πράγματα είναι πιο δύσκολα:

- ο χάρτης μπορεί να είναι άγνωστος ή ελλιπής,
- οι αισθητήρες μπορεί να έχουν θόρυβο,
- η θέση του ρομπότ μπορεί να είναι αβέβαιη,
- μπορεί να εμφανιστούν νέα εμπόδια,
- άνθρωποι ή άλλα ρομπότ μπορεί να κινούνται στον ίδιο χώρο,
- ένα δίκτυο επικοινωνίας μπορεί να καθυστερεί ή να αποτυγχάνει,
- ένα learning model μπορεί να κάνει λάθος,
- κάποιο robot σε ένα swarm μπορεί να στέλνει λανθασμένη ή κακόβουλη πληροφορία.

Το DynNav δημιουργήθηκε για να μελετήσει αυτά τα προβλήματα όχι ως «ειδικές περιπτώσεις», αλλά ως βασικά στοιχεία της αυτόνομης πλοήγησης.

Για αυτόν τον λόγο το project δεν αποτελεί μία απλή υλοποίηση του A* ή ενός ROS navigation stack. Αντίθετα, οργανώνει **26 ανεξάρτητες ερευνητικές συνεισφορές**, καθεμία από τις οποίες εξετάζει διαφορετικό πρόβλημα της ασφαλούς και αβέβαιης πλοήγησης.

Η λογική του project είναι:

> **Αν θέλουμε ένα ρομπότ να πλοηγείται στον πραγματικό κόσμο, δεν αρκεί να βρίσκει μια διαδρομή. Πρέπει να μπορεί να εκτιμά αβεβαιότητα, να αξιολογεί κίνδυνο, να διορθώνει αποτυχίες, να συνεργάζεται, να προστατεύεται και να εξηγεί γιατί πήρε μια απόφαση.**

# 2. Τι προσπαθεί να λύσει το DynNav

## 2.1 Το απλό πρόβλημα

Το πιο απλό πρόβλημα πλοήγησης είναι το εξής:

> Ένα ρομπότ βρίσκεται στο σημείο Α και πρέπει να φτάσει στο σημείο Β χωρίς να συγκρουστεί με εμπόδια.

Αν ο χάρτης είναι πλήρως γνωστός και στατικός, τότε ένας κλασικός αλγόριθμος σχεδιασμού διαδρομής μπορεί να βρει λύση. Για παράδειγμα, ο A* μπορεί να αναζητήσει πάνω σε ένα grid και να βρει μια διαδρομή από την αρχή μέχρι τον στόχο.

## 2.2 Το πραγματικό πρόβλημα

Στον πραγματικό κόσμο, όμως, το ρομπότ σπάνια έχει τέλεια πληροφορία. Η πλοήγηση γίνεται πιο δύσκολη για τους εξής λόγους:

### Αβεβαιότητα θέσης

Το ρομπότ δεν γνωρίζει ποτέ με απόλυτη ακρίβεια πού βρίσκεται. Η odometry συσσωρεύει σφάλματα, η IMU έχει θόρυβο, το LiDAR μπορεί να δώσει αβέβαιες μετρήσεις, και ο χάρτης μπορεί να μην ταιριάζει απόλυτα με το πραγματικό περιβάλλον.

### Άγνωστο ή μεταβαλλόμενο περιβάλλον

Το περιβάλλον μπορεί να αλλάζει ενώ το ρομπότ κινείται. Ένα εμπόδιο μπορεί να εμφανιστεί μετά τον αρχικό σχεδιασμό διαδρομής. Έτσι, η αρχική πορεία δεν είναι πάντα ασφαλής μέχρι το τέλος.

### Κίνδυνος

Η μικρότερη διαδρομή δεν είναι απαραίτητα η ασφαλέστερη. Μια πορεία μπορεί να είναι σύντομη αλλά να περνά δίπλα από εμπόδια, από περιοχές μεγάλης αβεβαιότητας ή από ζώνες όπου υπάρχει αυξημένος κίνδυνος αποτυχίας.

### Learning-based αστοχίες

Τα σύγχρονα συστήματα πλοήγησης χρησιμοποιούν συχνά machine learning. Όμως ένα νευρωνικό δίκτυο μπορεί να αποτύχει όταν δει κατάσταση διαφορετική από αυτές που είδε στην εκπαίδευση.

### Multi-robot ασυνέπειες

Όταν πολλά ρομπότ συνεργάζονται, πρέπει να συμφωνήσουν σε κοινές αποφάσεις. Αν κάποιο robot δίνει λάθος πληροφορία, είτε λόγω βλάβης είτε λόγω κακόβουλης συμπεριφοράς, το σύστημα πρέπει να παραμείνει ασφαλές.

## 2.3 Η απάντηση του DynNav

Το DynNav απαντά σε αυτά τα προβλήματα με μια modular προσέγγιση. Αντί να βασίζεται σε έναν μόνο planner, χωρίζει το πρόβλημα σε πολλά μικρότερα ερευνητικά προβλήματα:

- Πώς βελτιώνουμε τον A* με learned heuristics;
- Πώς εκτιμούμε αβεβαιότητα με EKF/UKF;
- Πώς σχεδιάζουμε διαδρομές λαμβάνοντας υπόψη worst-case risk;
- Πώς αποφεύγουμε καταστάσεις από τις οποίες δεν υπάρχει επιστροφή;
- Πώς μπαίνει το ρομπότ σε safe mode όταν αυξάνεται ο κίνδυνος;
- Πώς λαμβάνουμε υπόψη μπαταρία και συνδεσιμότητα;
- Πώς εξερευνά το ρομπότ περιοχές που προσφέρουν μέγιστη πληροφορία;
- Πώς ανιχνεύουμε spoofing ή ανωμαλίες αισθητήρων;
- Πώς συνεργάζονται πολλά ρομπότ χωρίς συγκρούσεις;
- Πώς ενσωματώνουμε foundation models, LLMs και VLMs χωρίς να θυσιάζεται η ασφάλεια;
- Πώς προστατευόμαστε από adversarial attacks;
- Πώς εξηγούμε γιατί απέτυχε ένα σύστημα;

Αυτός είναι ο λόγος που το DynNav είναι μεγάλο: δεν λύνει μόνο το πρόβλημα της διαδρομής, αλλά προσπαθεί να καλύψει πολλά σημεία αποτυχίας της πραγματικής αυτόνομης πλοήγησης.

# 3. Γιατί δεν είναι απλώς ένας planner

Ένας planner απαντά κυρίως στην ερώτηση:

> **Ποια διαδρομή πρέπει να ακολουθήσω;**

Το DynNav προσπαθεί να απαντήσει σε πολύ περισσότερες ερωτήσεις:

> **Μπορώ να εμπιστευτώ τον χάρτη;**

> **Πόσο σίγουρος είμαι για τη θέση μου;**

> **Αν πάω από εκεί, μπορώ να επιστρέψω;**

> **Αν εμφανιστεί εμπόδιο, μπορώ να επανασχεδιάσω γρήγορα;**

> **Αν ένας αισθητήρας μου λέει ψέματα, μπορώ να το καταλάβω;**

> **Αν ένα learned model κάνει λάθος, υπάρχει safety layer που θα το περιορίσει;**

> **Αν πολλά ρομπότ διαφωνούν, ποιο σχέδιο πρέπει να ακολουθήσουμε;**

Αυτή η φιλοσοφία κάνει το DynNav πιο κοντά σε ένα **research framework για safe autonomy** παρά σε ένα απλό path planning demo.

# 4. Κεντρική ιδέα του project

Η κεντρική ιδέα του DynNav είναι ότι η αυτόνομη πλοήγηση πρέπει να σχεδιάζεται με βάση τρεις αρχές:

## 4.1 Η αβεβαιότητα πρέπει να διατηρείται

Πολλά συστήματα παίρνουν αβέβαιες μετρήσεις και τις μετατρέπουν σε μία μοναδική τιμή. Για παράδειγμα, το ρομπότ μπορεί να υπολογίσει μία μόνο θέση, παρότι στην πραγματικότητα υπάρχει αβεβαιότητα γύρω από αυτή τη θέση.

Το DynNav αντιμετωπίζει την αβεβαιότητα ως βασική πληροφορία. Αν το ρομπότ δεν είναι σίγουρο για κάτι, αυτό πρέπει να επηρεάζει την απόφαση.

## 4.2 Ο κίνδυνος είναι πιο σημαντικός από το μέσο κόστος

Ένας planner που ελαχιστοποιεί μόνο το μέσο κόστος μπορεί να επιλέξει μια διαδρομή που φαίνεται καλή κατά μέσο όρο, αλλά έχει επικίνδυνα σενάρια αποτυχίας.

Το DynNav δίνει σημασία σε tail-risk metrics όπως το CVaR. Δηλαδή δεν κοιτά μόνο τι γίνεται συνήθως, αλλά και τι γίνεται στις χειρότερες περιπτώσεις.

## 4.3 Η μάθηση πρέπει να συνδυάζεται με δομή και ασφάλεια

Το project χρησιμοποιεί learning-based modules, όπως learned heuristics, PPO, curriculum RL, VLM/LLM agents και diffusion occupancy prediction. Όμως δεν αφήνει τη μάθηση ανεξέλεγκτη. Τη συνδυάζει με κλασικούς αλγορίθμους, formal safety shields και risk-aware constraints.

Αυτό είναι σημαντικό γιατί στην πραγματική ρομποτική ένα learning model μπορεί να βοηθήσει πολύ, αλλά δεν πρέπει να αποτελεί το μοναδικό επίπεδο απόφασης σε safety-critical εφαρμογές.

# 5. Συνολική αρχιτεκτονική

Το DynNav οργανώνεται σαν στοίβα υποσυστημάτων:

```text
ROS 2 Humble / TurtleBot3 / Gazebo / Nav2 / slam_toolbox
        ↓
Perception Layer
        ↓
Uncertainty Estimation / Mapping / Sensor Processing
        ↓
Planning Core
        ↓
Risk-Aware Planning / Belief-Space Planning / Learned A*
        ↓
Safety Layer
        ↓
STL Monitors / CBF Filters / Safe Mode / Returnability
        ↓
Learning Layer
        ↓
PPO / Curriculum RL / Federated Learning / Foundation Models
        ↓
Coordination & Security Layer
        ↓
Multi-Robot Planning / IDS / Adversarial Robustness / BFT Consensus
```

Σε απλά λόγια:

- το ROS 2 παρέχει τη βάση εκτέλεσης,
- το perception layer δίνει πληροφορίες για τον κόσμο,
- το uncertainty layer υπολογίζει πόσο αξιόπιστες είναι αυτές οι πληροφορίες,
- το planning layer βρίσκει πιθανές διαδρομές,
- το risk layer αξιολογεί πόσο επικίνδυνες είναι,
- το safety layer περιορίζει επικίνδυνες εντολές,
- το learning layer βελτιώνει την απόδοση,
- το coordination layer επιτρέπει συνεργασία πολλών ρομπότ,
- το security layer εξετάζει τι συμβαίνει όταν υπάρχουν αστοχίες ή επιθέσεις.

## Contribution 01 — Learned A* Heuristics

### Ποιο πρόβλημα υπήρχε;

Ο κλασικός A* είναι ένας αξιόπιστος αλγόριθμος αναζήτησης διαδρομής. Μπορεί να βρει βέλτιστη διαδρομή όταν η heuristic είναι σωστή και ο χώρος αναζήτησης είναι διαχειρίσιμος.

Το πρόβλημα εμφανίζεται όταν ο χάρτης μεγαλώνει ή όταν το ρομπότ πρέπει να κάνει συχνό replanning. Σε τέτοιες περιπτώσεις ο A* μπορεί να χρειαστεί να εξερευνήσει πολλούς κόμβους πριν βρει λύση. Αυτό αυξάνει τον χρόνο υπολογισμού.

Για ένα ρομπότ που κινείται σε δυναμικό περιβάλλον, ο χρόνος planning είναι σημαντικός. Αν ο planner καθυστερήσει, το ρομπότ μπορεί να συνεχίσει να κινείται με παλιά πληροφορία.

### Τι υλοποιήθηκε;

Στο module αυτό υλοποιήθηκε μια learning-augmented εκδοχή του A*. Η ιδέα είναι ότι η heuristic μπορεί να ενισχυθεί με μάθηση, ώστε να οδηγεί την αναζήτηση πιο γρήγορα προς χρήσιμες περιοχές του χάρτη.

Δεν αντικαθίσταται ο A*. Αντίθετα, ο A* παραμένει ο βασικός planner, αλλά η heuristic γίνεται πιο έξυπνη.

### Γιατί είναι σημαντικό;

Η προσέγγιση αυτή συνδυάζει το καλύτερο από δύο κόσμους:

- την αξιοπιστία ενός κλασικού graph-search planner,
- και την προσαρμοστικότητα ενός learned μοντέλου.

Αυτό είναι σημαντικό για το DynNav, γιατί το project θέλει learning where it helps, structure where it matters.

### Τι έδειξαν τα αποτελέσματα;

Σύμφωνα με το κύριο README του project, η learned heuristic μειώνει τις node expansions περίπου κατά **35%**.

### Γιατί προέκυψε αυτό;

Η μείωση των node expansions είναι λογική, γιατί το learned heuristic βοηθά τον planner να μην εξερευνά ισότιμα όλες τις πιθανές περιοχές. Αντίθετα, τον κατευθύνει προς κόμβους που είναι πιθανότερο να οδηγήσουν στον στόχο.

Αυτό δεν σημαίνει ότι το learned heuristic είναι πάντα τέλειο. Αν το περιβάλλον διαφέρει πολύ από αυτά που έμαθε το μοντέλο, η βελτίωση μπορεί να μειωθεί. Για αυτό το DynNav δεν βασίζεται αποκλειστικά στη μάθηση, αλλά την ενσωματώνει μέσα σε κλασικό planning.

## Contribution 02 — Uncertainty Estimation με EKF/UKF

### Ποιο πρόβλημα υπήρχε;

Ένα ρομπότ δεν γνωρίζει ποτέ με απόλυτη ακρίβεια τη θέση και την κατάστασή του. Οι αισθητήρες έχουν θόρυβο και οι μετρήσεις μπορεί να είναι ατελείς.

Αν το ρομπότ αγνοήσει αυτή την αβεβαιότητα, μπορεί να σχεδιάσει διαδρομή που φαίνεται ασφαλής στον χάρτη αλλά στην πράξη περνά επικίνδυνα κοντά από εμπόδια.

### Τι υλοποιήθηκε;

Το module αυτό χρησιμοποιεί EKF και UKF για εκτίμηση belief-state. Δηλαδή δεν κρατά μόνο μία εκτίμηση της θέσης, αλλά προσπαθεί να εκφράσει και την αβεβαιότητα γύρω από αυτή.

Το EKF είναι κατάλληλο για μη γραμμικά συστήματα που μπορούν να προσεγγιστούν μέσω γραμμικοποίησης. Το UKF χρησιμοποιεί sigma points και μπορεί να χειριστεί μη γραμμικότητες με διαφορετικό τρόπο.

### Γιατί είναι σημαντικό;

Το DynNav χρειάζεται uncertainty estimation γιατί πολλά από τα επόμενα modules βασίζονται σε αυτή την πληροφορία. Αν δεν γνωρίζουμε πόσο αβέβαιη είναι η κατάσταση του ρομπότ, δεν μπορούμε να κάνουμε σωστό risk-aware planning.

### Τι έδειξαν τα αποτελέσματα;

Το module αξιολογεί EKF/UKF belief-state για noisy sensors. Η βασική σημασία των αποτελεσμάτων είναι ότι η εκτίμηση αβεβαιότητας επιτρέπει στο σύστημα να λαμβάνει πιο προσεκτικές αποφάσεις σε σχέση με έναν planner που θεωρεί τη θέση του ρομπότ απόλυτα γνωστή.

### Γιατί προέκυψε αυτό;

Η βελτίωση προκύπτει επειδή το σύστημα πλέον δεν παίρνει αποφάσεις πάνω σε μία μόνο εκτίμηση. Αν η αβεβαιότητα αυξηθεί, αυτό μπορεί να επηρεάσει τον σχεδιασμό και να οδηγήσει σε πιο ασφαλείς επιλογές.

## Contribution 03 — Belief-Space & Risk Planning

### Ποιο πρόβλημα υπήρχε;

Ο κλασικός planning συνήθως θεωρεί ότι το ρομπότ γνωρίζει με ακρίβεια την κατάσταση του κόσμου. Αυτό δεν ισχύει σε πραγματικές συνθήκες.

Επιπλέον, πολλοί planners ελαχιστοποιούν το αναμενόμενο κόστος. Όμως μια διαδρομή με καλό μέσο κόστος μπορεί να έχει πολύ κακά worst-case σενάρια.

### Τι υλοποιήθηκε;

Το module αυτό εισάγει belief-space και risk planning. Η διαδρομή δεν αξιολογείται μόνο με βάση απόσταση ή αναμενόμενο κόστος, αλλά και με βάση τον κίνδυνο. Χρησιμοποιείται CVaR-optimized risk-weighted A*.

### Γιατί είναι σημαντικό;

Το CVaR δίνει έμφαση στις χειρότερες περιπτώσεις. Αυτό ταιριάζει σε ρομποτική πλοήγηση, γιατί ένα μικρό ποσοστό επικίνδυνων αποτυχιών μπορεί να είναι αρκετό για να κάνει ένα σύστημα μη αποδεκτό.

### Τι έδειξαν τα αποτελέσματα;

Τα αποτελέσματα του module δείχνουν τη χρησιμότητα του risk-weighted planning: το ρομπότ δεν επιλέγει απλώς τη φθηνότερη διαδρομή, αλλά λαμβάνει υπόψη περιοχές υψηλού κινδύνου.

### Γιατί προέκυψε αυτό;

Η συμπεριφορά αυτή προκύπτει επειδή το CVaR αλλάζει το κριτήριο βελτιστοποίησης. Ο planner δεν ρωτά μόνο «ποια διαδρομή είναι πιο σύντομη;», αλλά και «ποια διαδρομή αποφεύγει επικίνδυνα tail outcomes;».

---

## Contribution 04 — Irreversibility & Returnability

### Ποιο πρόβλημα υπήρχε;

Σε πολλά περιβάλλοντα, ένα ρομπότ μπορεί να μπει σε περιοχή από την οποία είναι δύσκολο ή αδύνατο να επιστρέψει. Αυτό μπορεί να συμβεί σε στενούς διαδρόμους, dead-ends, περιοχές χαμηλής συνδεσιμότητας ή περιοχές όπου το ρομπότ δεν έχει αρκετή πληροφορία για ασφαλή επιστροφή.

### Τι υλοποιήθηκε;

Το module αυτό εισάγει returnability constraints. Δηλαδή κάθε απόφαση αξιολογείται όχι μόνο με βάση το αν προχωρά προς τον στόχο, αλλά και με βάση το αν το ρομπότ διατηρεί δυνατότητα επιστροφής.

### Γιατί είναι σημαντικό;

## Contribution 26 — Swarm Consensus

### Ποιο πρόβλημα υπήρχε;

Σε ένα swarm, πολλά robots πρέπει να συμφωνήσουν σε κοινό σχέδιο. Αν κάποια robots είναι Byzantine, δηλαδή στέλνουν λανθασμένες ή κακόβουλες πληροφορίες, η απλή πλειοψηφία μπορεί να αποτύχει.

### Τι υλοποιήθηκε;

Το module υλοποιεί Byzantine fault-tolerant plan consensus με weighted median.

### Γιατί είναι σημαντικό;

Ένα multi-robot σύστημα πρέπει να παραμένει αξιόπιστο ακόμη και όταν μέρος των agents δεν συμπεριφέρεται σωστά.

### Τι έδειξαν τα αποτελέσματα;

Το κύριο README αναφέρει:

| Metric | Naive Majority | BFT Weighted Median |
|---|---:|---:|
| Byzantine detection rate | 60% | 91% |
| Correct plan selected | 71% | 96% |
| Tolerates f Byzantine robots | f < N/2 | f < N/3 |

### Γιατί προέκυψε αυτό;

Η βελτίωση προκύπτει επειδή το weighted median είναι πιο ανθεκτικό σε ακραίες ή κακόβουλες τιμές από μια απλή πλειοψηφία. Το σύστημα δεν εμπιστεύεται ισότιμα όλες τις προτάσεις, αλλά περιορίζει την επίδραση ύποπτων agents.

# 8. Συνολικά αποτελέσματα και ερμηνεία

Τα αποτελέσματα του DynNav δείχνουν ότι διαφορετικά modules βελτιώνουν διαφορετικές πλευρές της αυτόνομης πλοήγησης.

Δεν υπάρχει ένα μόνο metric που να περιγράφει όλο το project. Αντίθετα, κάθε contribution αξιολογείται με βάση το πρόβλημα που προσπαθεί να λύσει.

## 8.1 Learned A* και planning efficiency

Η μείωση των node expansions περίπου κατά 35% δείχνει ότι η learning-augmented heuristic μπορεί να κάνει τον A* πιο αποδοτικό. Αυτό είναι σημαντικό για dynamic rerouting, γιατί κάθε replanning πρέπει να γίνεται γρήγορα.

## 8.2 Safety Shields

Τα formal safety shields μειώνουν τις constraint violations από 4.2 ανά επεισόδιο σε 0.3, με path overhead μικρότερο από 8%. Αυτό είναι από τα πιο σημαντικά αποτελέσματα του project, γιατί δείχνει ότι μπορεί να προστεθεί formal safety χωρίς να καταστραφεί η αποδοτικότητα της πλοήγησης.

## 8.3 Swarm Consensus

Το BFT weighted median αυξάνει το correct plan selection από 71% σε 96% και το Byzantine detection από 60% σε 91%. Αυτό δείχνει ότι η ανθεκτικότητα σε προβληματικούς agents είναι κρίσιμη για multi-robot systems.

## 8.4 Federated Learning

Το federated learning πλησιάζει την centralized απόδοση, ειδικά μετά από αρκετούς γύρους. Η μικρή διαφορά είναι αναμενόμενη, γιατί τα δεδομένα είναι κατανεμημένα και όχι πλήρως συγκεντρωμένα.

## 8.5 Curriculum RL

Η αύξηση του success rate από 23% σε 61% δείχνει ότι το curriculum είναι πολύ σημαντικό σε δύσκολες RL αποστολές. Το agent μαθαίνει καλύτερα όταν η δυσκολία αυξάνεται σταδιακά.

# 9. Τι δείχνει συνολικά το DynNav

Το DynNav δείχνει ότι η αυτόνομη πλοήγηση δεν πρέπει να αντιμετωπίζεται ως ένα μόνο πρόβλημα εύρεσης διαδρομής.

Ένα πραγματικά αξιόπιστο navigation framework πρέπει να περιλαμβάνει:

- planning,
- uncertainty estimation,
- risk-aware decision making,
- safety verification,
- learning-based βελτιώσεις,
- anomaly detection,
- adversarial robustness,
- multi-robot coordination,
- explainability,
- semantic understanding.

Το σημαντικότερο συμπέρασμα είναι ότι κάθε module καλύπτει διαφορετικό failure mode. Αυτό κάνει το DynNav ενδιαφέρον ως research framework: δεν προσπαθεί να αποδείξει ότι ένας αλγόριθμος λύνει τα πάντα, αλλά ότι η ασφαλής αυτονομία απαιτεί συνδυασμό πολλών επιπέδων προστασίας και ευφυΐας.

# 11. Συμπέρασμα

Το DynNav είναι ένα μεγάλο ερευνητικό framework για δυναμική πλοήγηση και επανασχεδιασμό διαδρομής σε άγνωστα περιβάλλοντα.

Η βασική του αξία δεν βρίσκεται μόνο στο ότι περιλαμβάνει πολλούς αλγορίθμους, αλλά στο ότι οργανώνει αυτούς τους αλγορίθμους γύρω από ένα κεντρικό ερευνητικό ερώτημα:

> **Πώς μπορεί ένα ρομπότ να παραμένει ασφαλές όταν ο κόσμος, οι αισθητήρες, οι χάρτες, τα learning models και τα άλλα robots δεν είναι πλήρως αξιόπιστα;**

Τα 26 contributions δείχνουν διαφορετικές απαντήσεις σε αυτό το ερώτημα. Άλλα modules βελτιώνουν το planning, άλλα εκτιμούν αβεβαιότητα, άλλα προσθέτουν safety guarantees, άλλα εξετάζουν security, άλλα υποστηρίζουν multi-robot coordination και άλλα εισάγουν learning-based ή foundation-model-based δυνατότητες.

Συνολικά, το DynNav δείχνει ότι η ασφαλής αυτόνομη πλοήγηση δεν είναι ένα πρόβλημα ενός μόνο planner. Είναι ένα πολυεπίπεδο πρόβλημα που απαιτεί συνδυασμό κλασικής ρομποτικής, πιθανοτικής εκτίμησης, μηχανικής μάθησης, formal methods και robust distributed decision making.
