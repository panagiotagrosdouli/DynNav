# Contribution 10 — Human-Aware and Ethics-Guided Navigation

[![Module](https://img.shields.io/badge/Module-10-purple)](.) [![Type](https://img.shields.io/badge/Type-HRI%20%2F%20Ethics-blue)](.) [![Status](https://img.shields.io/badge/Status-Core%20Upgraded-brightgreen)](.)

## Plain-language summary

A robot should not treat every physically reachable path as acceptable.

It must also consider:

```text
Is this area private or restricted?
Is a human nearby?
Should the robot slow down or announce itself?
Is the operator confident enough to allow autonomy?
Is the language instruction clear enough to execute safely?
```

Contribution 10 adds a human-aware and ethics-guided policy layer for navigation. The upgraded version converts ethical zones, human proximity, operator trust, and language confidence into concrete planner actions.

---

## Research Question

> **RQ10:** Can autonomous navigation incorporate human preferences, operator trust, and ethical constraints in a transparent and auditable way?

This contribution studies:

- hard no-go zones,
- soft ethical zones,
- slow zones,
- announcement zones,
- human proximity adaptation,
- operator trust,
- language ambiguity,
- autonomy-level modulation.

---

## Motivation

A classical planner may find a path that is collision-free and short, but still unacceptable.

Examples:

- the path crosses a private area,
- the robot moves too fast near humans,
- an instruction is ambiguous,
- the operator does not trust the robot enough for full autonomy,
- the robot should announce itself before entering a shared space,
- a path is technically possible but socially inappropriate.

C10 makes these concerns explicit planner inputs.

## Conceptual Pipeline

```text
ethical zones + human proximity + operator trust + language confidence
      ↓
human-ethics policy
      ↓
planner action + speed limit + autonomy level + explanation
      ↓
execution, safe-mode trigger, or operator confirmation
```

## Ethical Zone Types

| Zone Type | Behaviour |
|---|---|
| `no_go` | Hard constraint: path is blocked |
| `slow_zone` | Reduce maximum velocity |
| `announce` | Announce robot presence and proceed cautiously |
| `avoid_if_possible` | Soft penalty in cost function |
| `none` | No ethical zone constraint |

---

## Planner Actions

| Action | Meaning |
|---|---|
| `NORMAL` | Proceed normally |
| `SLOW_DOWN` | Reduce speed and increase caution |
| `ANNOUNCE_AND_SLOW` | Announce presence and proceed slowly |
| `AVOID_ZONE` | Add penalty and avoid unless necessary |
| `BLOCK_PATH` | Reject the path |
| `ASK_OPERATOR` | Request human confirmation before continuing |

## Limitations

- The policy is rule-based and interpretable, not learned from human feedback.
- Human detection reliability is assumed.
- Ethical zone definitions are simplified.
- Real deployment should account for local policy, privacy requirements, social norms, and institutional rules.
- Language confidence is represented as a scalar; real language grounding requires richer ambiguity detection.

---

## Next Research Step

The strongest extension is preference learning under hard ethical constraints:

```text
human feedback → preference model → planner cost / constraints
```

This would allow the robot to adapt to specific operators and environments while preserving hard no-go and safety rules.
