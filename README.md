# DynNav: Dynamic Navigation Rerouting in Unknown Environments

![Status](https://img.shields.io/badge/results-Synthetic%20Demo-orange) ![Python](https://img.shields.io/badge/python-3.10%2B-blue)

DynNav is a working research prototype for dynamic rerouting in unknown and changing environments with risk-aware, uncertainty-aware, and recoverability-aware planning.

> Research question: How can an autonomous robot dynamically reroute in unknown and changing environments while reasoning about uncertainty, risk, recoverability, and mission safety?

## Working Synthetic Demo

```bash
pip install -r requirements.txt
python scripts/run_all.py
```

This generates deterministic **Synthetic Demo** outputs from code: `results/metrics/summary.json`, `metrics.csv`, `executed_trajectory.csv`, `reroute_events.csv`, `safety_events.csv`, `results/figures/*.png`, `assets/figures/*.png`, `assets/gifs/demo.gif`, `assets/videos/demo.mp4`, `results/videos/dynnav_demo.gif`, and `results/videos/dynnav_demo.mp4`.

No real-robot or state-of-the-art benchmark claim is made. Current quantitative outputs are synthetic grid-world demonstrations.

| ROS2/Nav2 | Prototype scaffold | `dynnav/ros2/README.md` |
| Real robot validation | Planned | not claimed |

Objective prototype cost:

`total_cost = path_cost + lambda_risk * risk + lambda_uncertainty * uncertainty - lambda_recoverability * recoverability`

Shortest-path navigation is insufficient when the shortest corridor is uncertain, dynamically blocked, close to obstacles, or hard to recover from.

## Limitations

The demo is deterministic and synthetic. ROS2/Nav2 files are scaffolds. Benchmark CSVs are generated by the included code and should be regenerated after algorithm changes.
